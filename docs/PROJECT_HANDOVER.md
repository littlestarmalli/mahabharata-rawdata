# Mahabharata Knowledge Graph вЂ” Project Documentation

## What This Project Is

This project takes a single **57 MB PDF file** ("The Mahabharata Set of 10 Volumes.pdf" вЂ” 3,723 pages, Hindi text with English footnotes) and extracts a complete, structured knowledge graph from it. The output is **3 interconnected JSON files** (characters, locations, timeline) plus extracted raw text, all generated by a modular Python pipeline.

| Item | Details |
|------|---------|
| **Source PDF** | `The Mahabharata Set of 10 Volumes.pdf` вЂ” 57 MB, 3,723 pages |
| **Content** | 10 volumes of the Mahabharata in Hindi with English footnotes |
| **Schema spec** | `charactor-creator.md` вЂ” defines the target 3-file JSON schema |

### Step 1: PDF Extraction (`extract/pdf_parser.py`)
- Uses PyMuPDF (`fitz`) to parse all 3,723 pages
- Detects volume boundaries from page headers
- Separates each volume into 3 files: chapters, footnotes, table of contents
- Font-size threshold distinguishes body text from footnotes

### Step 1B: Manual OCR Fixes (`fixes/ocr_fixes.py`)
- Data-driven: each fix is a tuple (volume, file_type, old_text, new_text, description)
- 4 specific fixes for volumes 5 and 10

### Step 2: Section Headers (`extract/section_headers.py`)
- Injects matching headers into footnote files for traceability

### Step 3: Verify Footnote Matching (`extract/verify.py`)
- **Achievement: 100% match rate вЂ” 16,734 out of 16,734 footnotes matched**

### Step 4: Build Knowledge Graph

#### Phase 1: Core Characters (`data/characters.py`)
- **174 hardcoded `add()` calls** + Kaurava loop (100 sons of Dhritarashtra)
- ~35 major characters have expanded fields: Skills, Divine_Weapons, Titles, Traits, Character_Arc, Key_Relationships

#### Phase 2: Auto-extraction (`auto/name_extractor.py`)
- Regex-based extraction of names and relationships from footnotes and chapter text
- Patterns: "son of", "wife of", "also known as", "King/Prince/Sage Name"

#### Phase 3: Merge & Resolve (`merge/resolver.py`)
- MERGE_MAP maps spelling variants to canonical keys
- Deduplicates, removes noise, validates references

#### Phase 4: Output characters.json (`output/builder.py`)
- Auto-derives Lineage (Grandfather/Great_Grandfather from father chain)
- 525 characters, 163 KB

#### Phase 5: Locations (`data/locations.py`)
- 52 locations, 47 with real-world coordinates, 5 mythical
- Haversine distance computation (500 km threshold)

#### Phase 6: Timeline (`data/timeline.py`)
- 58 events, 4 eras, Year -80 to +36 (0 = Kurukshetra War)

#### Phase 7: Cross-linking (`output/crosslink.py`)
- Adds `Events_Occurred` to locations, `Major_Events` to characters

#### Phase 8: Validation (`output/validator.py`)
- Validates all @refs, timeline order, year monotonicity
- **Result: Zero validation warnings**

## Cross-Reference System

All three files are interconnected using `@` prefix keys:

| From | Field | Links To |
|------|-------|----------|
| Character | `Father`, `Mother`, `Spouse` | Other characters (`@char_key`) |
| Character | `Kingdom`, `Important_Locations` | Locations (`@loc_key`) |
| Character | `Major_Events` | Timeline (`@evt_key`) |
| Location | `Ruler`, `Residents` | Characters (`@char_key`) |
| Location | `Events_Occurred` | Timeline (`@evt_key`) |
| Timeline | `Participants` | Characters (`@char_key`) |
| Timeline | `Location` | Locations (`@loc_key`) |
| Timeline | `Related_Events` | Other events (`@evt_key`) |

The validation in Phase 8 ensures **every single reference** resolves to an existing entry. Zero broken links.

## Important Technical Notes

- **All character data is hardcoded** (user explicitly chose this over AI extraction for accuracy)
- The `add()` function handles deduplication вЂ” calling it twice for the same character merges data
- The `ensure()` function creates minimal stubs for characters referenced but not fully defined
- Spouse model: `list[tuple(str, str)]` internally вЂ” `(spouse_key, relation_type)`
- Lineage (Grandfather/Great_Grandfather) is auto-derived by walking the Father chain вЂ” not hardcoded
- The `@` prefix in keys is a convention for cross-reference IDs (not Python syntax)
- Distance computation uses haversine formula with R = 6,371 km; threshold 500 km for "nearby"
- PowerShell in Windows has issues with f-strings containing quotes вЂ” use helper .py files for complex commands
